图形化版贪吃蛇概要设计


1. 问题描述
图形化版贪吃蛇是一个具有图形化用户界面的交互式游戏程序，需要实现窗口绘制、键盘控制、实时计分与游戏状态展示。
本版本由 3 人小组合作开发，需要完成需求分析、界面设计、编码实现和测试工作。除了基本的单人游戏与计分功能外，我们还设计了双人对战以及人机对战模式等以增加游戏趣味性。
2. 需求分析
2.1 功能需求
基础功能：
创建游戏窗口。
绘制 35×35 游戏地图。
绘制蛇头、蛇身、食物、墙和障碍物。
支持键盘控制蛇移动。
支持实时计分。
支持碰撞检测与游戏结束。
支持重新开始游戏。
拓展功能：
开始菜单界面。
暂停 / 继续。
游戏结束界面。
双人对战模式。
人机对战模式。
人机对战模式中的难度选择。
单人模式中最高分记录。
食物随机生成。
加速游戏进程（按蛇的默认移动方向）。
背景音乐与音效。
2.2 输入 / 输出需求
输入：
键盘方向输入。
加速、暂停、重开、退出按键。
可选的人机对战难度。
输出：
游戏窗口画面。
当前分数。
当前游戏状态。
游戏结束提示。
3. 界面设计
3.1 主界面布局
主界面应包含：
左侧或中央：游戏地图区域。
右侧和或下方：分数、长度、状态、操作提示。
游戏结束时：显示最终分数和重新开始提示。
窗口布局大致为：
┌──────────────────────┬──────────────┐
│                      │  当前分数     │
│    35 × 35 地图      │  最高分       │
│                      │  当前状态     │
│                      │  操作提示     │
└──────────────────────┴──────────────┘
3.2 显示要求
地图格子大小固定（CELL_SIZE = 24px），避免移动时画面抖动。
蛇头和蛇身应当明显区分（头部带蛇眼，身体节段渐缩 + 颜色渐深）。
食物应当醒目（红色圆形 + 辉光效果）。
墙与障碍物应能直观看出不可通过。（用黑色和灰色来表现）
3.3 操作提示
界面中可以显示简短的操作提示：
W / A / S / D 或方向键：移动。
Space：暂停 / 继续。
R：重新开始。
Shift / F：加速（4 倍速）。
Esc：退出 / 返回菜单。

4.2 架构设计
系统采用”场景驱动 + 模块分层”架构：
顶层：main.cpp 负责窗口生命周期与场景调度
场景层：scene.h/cpp 管理菜单和游戏两种场景的状态流转
逻辑层：game.h/cpp 持有全部游戏状态，实现蛇移动、碰撞检测、食物/障碍物生成
AI 层：ai.h/cpp 基于 BFS 实现 AI 对手的方向决策（仅人机模式调用）
渲染层：render.h/cpp 负责全部图形绘制，只读 game 状态
音频层：audio.h/cpp 负责 PCM 音效的代码合成与播放
输入层：input.h/cpp 封装 raylib 键盘 API
数据流方向：
键盘输入 → scene（方向变更、暂停、重开）
        → game（状态更新：移动 + 碰撞 + 增长）
        → render（读取状态绘制画面）+ audio（触发音效）

6. 全局数据结构设计
6.1 坐标点 Point
typedef struct point {
    int row, col;
} Point;
用于表示蛇身各节在网格中的行列坐标。全部游戏坐标均以 (row, col) 序表示，row 范围 0 ～ map_size - 1，col 范围 0 ～ map_size - 1。
6.2 游戏状态 GameState
GameState 是所有游戏模式共享的核心状态结构体，定义于 game.h：
typedef struct gamestate {
    // ──── 地图 ────
    char map[MAX_MAP_SIZE][MAX_MAP_SIZE];   // 40×40 字符网格（最大容量）
    int map_size;                           // 实际地图尺寸（35）

    // ──── 玩家蛇（P1 / 经典模式唯一玩家） ────
    Point snake[MAX_SNAKE_LEN];     // 蛇身坐标数组（索引 0 = 头，length-1 = 尾）
    int length;                     // 当前蛇身长度
    int score;                      // 当前分数
    int move_count;                 // 累计移动步数
    int grow_step;                  // 自动成长间隔（默认 32）
    Direction direction;            // 当前方向枚举
    DeathReason death_reason;       // 死亡原因枚举
    bool started;                   // 游戏是否已开始（首次方向输入后置 true）
    bool paused;                    // 是否暂停
    bool game_over;                 // 游戏是否结束
    GameMode mode;                  // 当前游戏模式枚举
    int food_eaten;                 // 吃食物累计次数
    int pending_growth;             // 待兑现的击杀成长点数
    int high_score;                 // 经典模式历史最高分（文件持久化）

    // ──── 第二蛇（PvP 的 P2 / Battle 的 AI，复用 ai_snake 字段） ────
    Point ai_snake[MAX_SNAKE_LEN];
    int ai_length;
    Direction ai_direction;
    bool ai_is_alive;
    int ai_score;
    int ai_food_eaten;
    int ai_pending_growth;
    AIDifficulty ai_difficulty;
    DeathReason ai_death_reason;

    // ──── P2 方向存储（仅 PvP 模式使用） ────
    Direction p2_direction;
} GameState;
设计要点：
ai_snake 字段在 PvP 模式下复用为玩家 2 蛇身，在 Battle 模式下为 AI 蛇身，避免了重复定义。
pending_growth 用于实现击杀奖励机制：击杀方获得对方当前全长度的成长点数，在后续若干次移动中逐步消费（每步消费 1 点，吃食物和自动成长的回合不消费）。
地图字符编码采用单字符表示法：# = 墙，. = 空地，O = 障碍物，F = 食物，H = P1 蛇头，B = P1 蛇身，A = P2/AI 蛇头，C = P2/AI 蛇身。
snake[] 数组采用队列式存储：头在索引 0，尾在索引 length - 1；移动时整体右移一位，新头写入索引 0。

8. 高层算法设计
8.1 蛇移动通用算法
输入：GameState, 蛇身数组 body[], 蛇长 *len, 方向 dir, 分数 *score, pending_growth*
前提：game->started && !paused && !game_over

1.  计算 next = (old_head.row + dr[dir], old_head.col + dc[dir])
2.  判定 will_eat = (map[next] == 'F')
3.  判定 auto_grow = ((move_count + 1) % grow_step == 0)
4.  判定 has_pending = (pending_growth > 0)
5.  effective_grow = will_eat || auto_grow || has_pending
6.  调用 can_enter_cell(next, effective_grow, body, *len) 做碰撞检测
    - 若碰撞 → game_over = true，death_reason 赋值，return
7.  若不增长 → 清除尾部 map 标记（置为 '.'）
8.  若增长 → *len += 1（上限 MAX_SNAKE_LEN）
9.  若有 pending 且非吃食物非自动增长 → pending_growth -= 1
10. 旧头变身体（body_char），数组右移一位，新头写入 body[0]
11. 新头位置 map 标记为 head_char
12. move_count++
13. 若吃到食物 → *score += 10，*food_eaten++，随机放置新食物
增长优先级：pending_growth 只有在非吃食物、非自动增长的回合才被消费，确保三个增长来源不重叠。
8.2 碰撞检测算法（can_enter_cell）
输入：GameState*, 目标坐标 next, 是否会增长 will_grow, 蛇身数组, 蛇长
输出：bool（可通行），DeathReason（死亡原因）

规则（按优先级）：
- map[next] == '#'       → DEATH_WALL
- map[next] == 'O'       → DEATH_OBSTACLE
- map[next] == 'B'/'C'   → 若 !will_grow 且 next == body[length-1] → 允许（尾部即将移走）
                           否则 DEATH_BODY / DEATH_ENEMY_BODY
- map[next] == 'H'/'A'   → DEATH_HEAD_ON（对方蛇头）
- 其他                    → 允许
允许通过自身即将移除的尾部这一特殊处理，保证了蛇在非增长回合可以沿自身方向无冲突移动。
8.3 对战模式碰撞决议（PvP / Battle 共用）
1. 同时计算 P1_next 和 P2_next
2. 检查碰撞（优先级顺序）：
   a. is_same_point(P1_next, P2_next)
      → 双方均死（DEATH_HEAD_ON）
   b. P1_next 在 P2 身体上
      → P1 死，P2 获 P1 长度奖励（pending_growth += game->length）
   c. P2_next 在 P1 身体上
      → P2 死，P1 获 P2 长度奖励（pending_growth += game->ai_length）
   d. P1_next 撞墙 / 障碍物 → P1 死
   e. P2_next 撞墙 / 障碍物 → P2 死
   f. P1_next 撞自身 → P1 死
   g. P2_next 撞自身 → P2 死
3. 执行结果：
   - 双方均死 → 游戏结束
   - 一方死 → 存活方执行移动，游戏结束（Battle：AI 死则 respawn）
   - 双方活 → 分别执行 apply_snake_move()
4. 确保地图始终保持 ≥ 2 个食物
碰撞优先级保证了：头碰头同时死亡 > 互撞身体按先检查的为准 > 各自独立碰墙/碰自身。
8.4 AI 寻路算法
AI 系统的核心是 BFS 寻路，封装在 ai.cpp 中，通过 ai_choose_direction() 按难度分发。
核心数据结构：
int dist[MAX_MAP_SIZE][MAX_MAP_SIZE] — BFS 距离矩阵，-1 表示不可达
Node q[MAX_SNAKE_LEN] — BFS 队列（静态数组，避免动态分配）
四个方向常量数组：AI_DR[4] = {-1, 0, 1, 0}，AI_DC[4] = {0, -1, 0, 1}，AI_DIR[4] = {UP, LEFT, DOWN, RIGHT}
三级难度策略对比：
安全检测算法（Hard 难度，ai_can_reach_tail_after_move）：
输入：GameState*, 候选方向 d
输出：bool（移动后能否到达尾部）

1. 复制当前 map 到 sim[MAX_MAP_SIZE][MAX_MAP_SIZE]
2. 判定 will_grow = 此步会吃食物 || 此步是自动成长步
3. 若 !will_grow → sim[尾部] = '.'（模拟尾移除）
4. sim[旧头] = 'C'，sim[新头] = 'A'（模拟移动后的地图）
5. 计算新尾部坐标：
   - 若 will_grow → 旧尾部不变
   - 否则 → 取 body[len-2]（原尾部前一个元素）
6. BFS 搜索：新头 → 新尾部
7. 返回是否可达
该算法保证了 Hard 难度 AI 在蛇身较长时不会走入死胡同。
Easy 难度特殊说明：Easy 难度不使用 BFS 计算精确最短路径，而是用曼哈顿距离 |row - food_row| + |col - food_col| 评估四个相邻格，选择距离食物最近的方向。由于曼哈顿距离忽略了障碍物和蛇身的阻挡，Easy AI 经常”看起来在想但又不完全想”——这符合 Easy 难度的设计意图。
8.5 食物生成算法
1. 遍历 map，收集所有 '.' 单元格到 cells[] 数组
2. 若 cells 数量为 0 → game_over = true，death_reason = DEATH_FULL_MAP，return
3. idx = rand() % count
4. map[cells[idx]] = 'F'
食物生成在每步移动后进行，确保地图上始终有食物。在 PvP / Battle 模式中，额外维护至少 2 个食物的约束。
8.6 障碍物生成算法
输入：GameState*, 期望障碍物数量 count

1. 收集所有内部空地格（不含边缘墙）到 cells[]，总数 total
2. 上限 = total / 4（不超过空地格数的 25%，保证可玩性）
3. count = min(count, 上限)
4. 若 count ≤ 0 → 返回
5. 对 i = 0 到 count-1：
   j = i + rand() % (total - i)
   map[cells[j]] = 'O'
   swap(cells[i], cells[j])
该算法保证：障碍物随机分布、不重复、数量可控、不会导致地图不可玩。
8.7 AI 重生算法（Battle 模式）
1. 清除 AI 旧蛇身的地图标记（map 中 'A'、'C' → '.'）
2. 收集所有空地格到 cells[]
3. 尝试最多 200 次：
   随机选起始位置 start
   检查 start 纵向连续 3 格是否全为 '.'
   若是 → 放置 3 节竖直蛇身，方向 UP，成功
4. 若 200 次未成功 → 保底：随机放 1 节蛇身
5. 重置 AI 分数、吃食物计数、pending_growth 为 0
6. 确保地图 ≥ 2 个食物
8.8 程序化音效合成
全部音效由代码合成 PCM 波形，参数为 22050Hz / 16-bit / 单声道，通过 raylib 的 LoadSoundFromWave 加载。使用线性衰减包络 1.0 - t/duration 避免播放结束时的爆音。
8.9 中文字体加载策略
1. 构造字体路径列表：["C:/Windows/Fonts/simhei.ttf", "C:/Windows/Fonts/msyh.ttc"]
2. 构造所需码点数组：
   - ASCII 32-126（英文 + 数字 + 符号）
   - 游戏用到中文字符（从源码提取，约 100+ 字，UTF-8 → 码点转换）
3. 依次尝试 LoadFontEx(font_path, 48, codepoints, cp_count)
4. 若 glyphCount > 0 → 成功，break
5. 若全部失败 → 回退 GetFontDefault()
只加载游戏实际用到的中文字符码点（而非完整字库），显著减小字体纹理的内存占用。
