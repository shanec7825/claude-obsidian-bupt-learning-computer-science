# 第〇章：环境配置

> **学习目标**：装好 Visual Studio 2022 Community 和 EasyX 图形库，跑通第一个图形程序。
> **总时长**：约 3 小时（2 课时）
> **难度**：零基础友好，跟着做就能成

---

欢迎！这是整个课程的第一步。在你写出贪吃蛇之前，得先把"厨房"搭好——装好写代码需要的两个工具：**Visual Studio 2022 Community**（写代码的地方）和 **EasyX**（让程序能画图的工具箱）。

这一章没有编程经验要求。你唯一的任务就是跟着步骤走，每一步我都会解释"为什么要这么做"，而不是只告诉你"点这里"。如果中间卡住了，翻到最后的 FAQ 部分，大概率能找到答案。

> **停下来想一想**：你以前装过其他软件吗？装软件本质上就是三个动作——下载一个安装包、双击运行、一路点"下一步"。装开发工具比装普通软件多一步：你要手动勾选需要的"组件"（可以理解为软件的功能模块）。别紧张，下面我会用截图级的描述告诉你每一步该选什么。

---

## 0.1 安装 Visual Studio 2022 Community（约 40 分钟）

### 0.1.1 什么是 Visual Studio？

**Visual Studio**（简称 VS）是一个 **IDE**（Integrated Development Environment，集成开发环境）。你可以把它想象成一个"写代码的 Word"——有打字的地方、有保存按钮、有自动提示，而且它还能帮你把写好的代码翻译成电脑可以运行的程序。这个"翻译"的过程叫**编译**（Compile），翻译工具叫**编译器**（Compiler）。

我们用的是 **Community 版**：免费，个人开发完全够用。

### 0.1.2 下载

1. 打开浏览器，搜索 "Visual Studio 2022 Community 下载"，或者直接访问 `https://visualstudio.microsoft.com/zh-hans/downloads/`
2. 你会看到微软的下载页面。页面上有三个大的下载按钮，左边是 Community，中间是 Professional，右边是 Enterprise。
3. 点击左边的 **Community** 按钮下面的"免费下载"——你会下载到一个叫 `VisualStudioSetup.exe` 的文件（大约 3-4 MB）。

> **常见坑 #1**：不要下成 Visual Studio Code！VS Code 是一个轻量编辑器，和我们需要的 VS 是两样东西。判断方法：如果你打开的安装界面是深色背景、带大图标列表的，这是对的；如果是一个小小窗口让你选安装路径的，那是 VS Code，你下错了。

### 0.1.3 安装

1. 双击 `VisualStudioSetup.exe`。
2. 安装程序先会自解压，出现一个叫 **Visual Studio Installer** 的窗口。
3. 自解压完成后，你会看到主安装界面。屏幕上方写着"Workloads"（工作负载），下面是一排带图标的选项，比如：
   - ".NET 桌面开发"（一个显示器上面有窗口的图标）
   - "使用 C++ 的桌面开发"（一个齿轮+窗口的图标）——**这就是我们要选的**
   - "ASP.NET 和 Web 开发"
   - 还有很多其他选项……
4. **勾选"使用 C++ 的桌面开发"**。勾选后，右侧面板会显示"安装详细信息"。

   右侧面板列表分为三块："必选项"、"可选"和"单个组件"。对初学者来说，保持默认就够用了。但我建议你展开"单个组件"标签页，确认下面这几项被勾上了（它们可能已经被默认勾选）：
   - **MSVC v143 - VS 2022 C++ x64/x86 生成工具**——这是编译器的核心，没有它你的代码编译不了
   - **Windows 11 SDK**（或 Windows 10 SDK）——这是 Windows 的开发工具包，你的程序需要它才能在 Windows 上运行
   - **用于 Windows 的 C++ CMake 工具**——暂时用不到，但装了没坏处
   - **C++ AddressSanitizer**——可选，装了以后能帮你找到程序里的 bug

   > **常见坑 #2**：很多人在这里只勾了"使用 C++ 的桌面开发"的大框，但没有展开确认单个组件。有时候安装程序不会默认装上 Windows SDK，导致后面编译时报错。花两分钟确认一下，省得后面折腾。

5. 确认安装位置。默认安装在 C 盘。如果你 C 盘空间不太够（少于 20GB 空闲），可以点下方的"安装位置"切换标签页，把安装路径改到其他盘。不过 IDE 本体还是会装在 C 盘，你能改的只是工作负载组件和 SDK 的位置。

6. 点击右下角的 **"安装"** 按钮。

7. 等。下载和安装总共需要 20-40 分钟，取决于你的网速。安装程序会先下载所需文件（你会看到进度条下面的下载速度和剩余时间），然后自动开始安装。你不需要一直盯着屏幕。

> **动手试一下**：趁下载的时间，去接杯水，站起来活动一下。如果你愿意，可以搜一下"C++ Hello World"，提前感受一下我们要写的东西长什么样。但看不懂也没关系，下一节我们就会手把手写。

### 0.1.4 安装完成后的验证

安装完成后，Installer 会提示"已安装"。点击"启动"按钮。

第一次启动 Visual Studio 时：
- 会要求你登录微软账号。**没有账号可以点"以后再说"跳过**，不影响使用。
- 接下来会让你选主题：深色、浅色、蓝色。选你看着顺眼的就行，随时可以改。
- 然后是一个"开发设置"的下拉框：选择 **"使用 C++ 的桌面开发"**。

现在你应该看到 Visual Studio 的主界面了：顶部有菜单栏（文件、编辑、视图……），左侧可能是一个"开始"窗口，右侧空着。

> **验证步骤**：点菜单栏的"文件" -> "新建" -> "项目"。在弹出的窗口里搜索"C++"，你应该能看到"控制台应用"或"空项目"等模板选项。如果能看到，说明 VS 装好了。

**如果新建项目窗口里没有 C++ 选项**：回到 Visual Studio Installer，找到 Visual Studio 2022 那一行，点"修改"，重新勾选"使用 C++ 的桌面开发"，再点"安装"。

## 0.2 安装 EasyX 图形库（约 20 分钟）

### 0.2.1 什么是 EasyX？

**EasyX** 是一个给 C++ 用的 **图形库**（Graphics Library）。刚才装的 Visual Studio 只能写"黑窗口"程序——你运行后看到一个黑底白字的命令行窗口。但我们要做的是贪吃蛇游戏，需要画蛇、画食物、画网格。EasyX 就是让这个"黑窗口"变成可以画图的"画布"的工具。

为什么选 EasyX？因为它专为初学者设计，画一个圆的代码只需要一行：

```cpp
circle(100, 100, 50);  // 在坐标 (100, 100) 画一个半径为 50 的圆
```

对比一下用 Windows 原生 API 画圆的代码（大概 40 行），EasyX 简单了不止一个数量级。

### 0.2.2 下载和安装

1. 打开浏览器，搜索 "EasyX 官网"，或者直接访问 `https://easyx.cn/`
2. 页面顶部有一个大大的 **"下载 EasyX"** 按钮，点击。你会下载到一个以 `.exe` 结尾的安装文件（例如 `EasyX_2023xxxx.exe`，数字代表发布日期）。
3. 在下载 EasyX 之前，建议先关闭 Visual Studio。不关也行，但安装程序会提示你关闭——它在写文件到 VS 的目录里，关掉 VS 可以确保不出错。

> **常见坑 #3**：杀毒软件（如 Windows Defender、360 等）可能会拦截 EasyX 的安装程序，因为 EasyX 安装时会向 VS 的安装目录写入文件和注册表信息。如果遇到拦截，点"允许"或"仍然运行"。EasyX 官网下载的是安全文件——它被超过 50 万用户使用多年，没有任何安全问题。如果你真的不放心，可以在 VirusTotal 网站上上传文件检测。

4. 双击下载的 EasyX 安装程序。
5. 你会看到一个简洁的安装窗口，上面写着 "EasyX 安装程序"，下面是你电脑上检测到的 VS 版本列表。每一行 VS 版本前面有一个复选框，默认全部勾选。
6. **如果你装的是 VS 2022**，确保 "Visual Studio 2022" 那一行的复选框是勾上的。如果没有显示 VS 2022，说明：
   - 你的 VS 还没装完（回去看 0.1）
   - 或者你装的是 VS 2022 Preview 版而非正式版——EasyX 可能检测不到 Preview 版。解决办法是装正式版。
7. 点击 **"安装"** 按钮。几秒钟就完成了。
8. 看到"安装成功"的提示后，关掉安装程序。

### 0.2.3 验证 EasyX 安装

打开 Visual Studio，创建一个项目来验证：

1. 点"创建新项目"（或 "文件" -> "新建" -> "项目"）
2. 搜索框中输入"空项目"，选择 **"空项目 (.NET Framework)"** 或 **"空项目"**（C++ 的类型）
3. 给项目起个名字，比如 `EasyXTest`，选一个存放位置，点"创建"
4. 在右侧"解决方案资源管理器"中，右键点击"源文件" -> "添加" -> "新建项"
5. 选择"C++ 文件(.cpp)"，命名 `main.cpp`，点"添加"
6. 在打开的编辑窗口中输入以下代码：

```cpp
#include <graphics.h>

int main() {
    initgraph(640, 480);      // 创建 640x480 像素的绘图窗口
    circle(320, 240, 100);    // 在窗口中心画一个半径 100 的圆
    getchar();                // 等待用户按任意键再关闭窗口
    closegraph();             // 关闭绘图窗口
    return 0;
}
```

7. 点击顶部工具栏的绿色三角按钮（"本地 Windows 调试器"），或者按 `F5`。
8. 如果你看到一个深色背景的窗口，中间画着一个白色的圆——**恭喜，环境配置全部完成！**

> **常见坑 #4**：如果编译报错 `无法打开包括文件: "graphics.h": No such file or directory`，说明 EasyX 没有正确安装到 VS 的 include 目录。解决方法：
> 1. 重新运行 EasyX 安装程序
> 2. 确认 VS 2022 前面的复选框已勾选
> 3. 如果有多个 VS 版本（比如同时装了 2019 和 2022），确保你对应的版本都被勾上
> 4. 如果还不生效，以管理员身份运行 EasyX 安装程序

> **常见坑 #5**：如果编译通过了但运行报错，报一些类似 `main` 重复定义的错——检查你创建项目时选的是不是"空项目"。如果选了"控制台应用"，VS 会自动生成一个 `main`，和你的代码冲突了。删掉自动生成的那个文件，只保留自己写的就行。

## 0.3 第一个程序：Hello Red Circle（约 40 分钟）

环境跑通了，我们现在来认真写自己的第一个程序。前面那个画白圆的程序只是验证用的，我们不满足于只会"跑别人给的代码"。这一节，我们要搞懂这个程序的每一行在干什么，并做出自己的修改。

### 0.3.1 创建一个新项目

> 这一步你应该已经做过了，所以我们快速过一遍：新建空项目 -> 添加 main.cpp -> 开始写代码。习惯就好——每次写新东西，都是从这三步开始。

### 0.3.2 写出一个带颜色的圆

在上一步的 `main.cpp` 里，把代码改成下面这样（注意：中文那行是注释，不需要敲进去，但如果你敲了也一样能跑）：

```cpp
// 第 0 行：引入 EasyX 图形库的头文件
#include <graphics.h>

// 第 1 行：引入输入输出标准库，让我们能用 getchar() 等待按键
#include <conio.h>

int main() {
    // 第 2 步：创建一个 800x600 的绘图窗口
    initgraph(800, 600);

    // 第 3 步：设置背景色为黑色（其实就是清屏为黑色）
    setbkcolor(BLACK);
    cleardevice();

    // 第 4 步：设置画笔颜色为红色
    setcolor(RED);

    // 第 5 步：设置填充颜色为黄色
    setfillcolor(YELLOW);

    // 第 6 步：画一个圆（圆心 x=400, 圆心 y=300, 半径 r=150）
    //          fillcircle = 填色 + 圆，画的是实心圆
    fillcircle(400, 300, 150);

    // 第 7 步：在屏幕上显示一段文字
    settextcolor(WHITE);                           // 文字颜色设为白色
    settextstyle(24, 0, _T("微软雅黑"));            // 字号 24，字体 微软雅黑
    outtextxy(280, 280, _T("Hello Red Circle!"));  // 在 (280, 280) 位置显示文字

    // 第 8 步：等待用户按任意键
    getchar();

    // 第 9 步：关闭图形窗口，释放资源
    closegraph();

    return 0;  // 告诉操作系统：程序正常结束
}
```

点 `F5` 运行。

你应该看到一个 800x600 的窗口：**黑色背景**上有一个**黄色实心圆**，圆的外圈是**红色**，圆的中间写着白色的"Hello Red Circle!"。

按任意键，窗口关闭。

### 0.3.3 逐行解释——每一行在做什么

你现在可能一脸懵：`initgraph` 是什么？`#include` 又是什么？别急，我们一行一行拆开看。

---

**`#include <graphics.h>`** 和 **`#include <conio.h>`**

以 `#` 开头的叫**预处理指令**（Preprocessor Directive）。`#include` 的作用是"把另一个文件的内容复制粘贴到这里"。`graphics.h` 是 EasyX 提供的一个**头文件**（Header File），里面写好了 EasyX 所有函数的"目录"——告诉 VS 你可以用哪些函数（比如 `circle`、`initgraph`）。`conio.h` 是 C++ 标准的头文件，给了我们 `getchar()` 这个能在按任意键时才继续往下走的函数。

简单类比：头文件就像一本菜谱的目录。当你写 `fish()` 的时候，编译器翻到目录一看——"哦，`fish` 在 EasyX 那本菜谱的第 42 页，我去那里找怎么做的"。

---

**`int main() { ... }`**

每一个 C++ 程序都必须有一个 `main` 函数。`main` 是程序的**入口**——当你双击程序运行时，操作系统会自动找到 `main` 并从这里开始执行。

`int` 表示这个函数会给操作系统返回一个整数（0 表示"一切正常"），`()` 表示这个函数不需要外部传入参数。

---

**`initgraph(800, 600);`**

这个函数（Function）的意思是"**初始化图形窗口**"（Initialize Graphics Window）。括号里的两个数字分别是窗口的宽和高，单位是**像素**（Pixel）。

第 4 节会详细讲什么是像素。现在你只需要理解：`800, 600` 意味着我们创建了一个 800 列 x 600 行的小方格组成的画布。

---

**`setbkcolor(BLACK);` + `cleardevice();`**

- `setbkcolor` = Set Background Color，设置背景颜色。`BLACK` 是一个系统预定义好的颜色常量（Color Constant）。
- `cleardevice` = Clear Device，用当前背景色把整个窗口涂一遍。

这两行通常一起出现：先告诉系统"我想要的背景色是什么"，然后执行清屏。

---

**`setcolor(RED);`** 和 **`setfillcolor(YELLOW);`**

- `setcolor` 设置**线条颜色**（笔画色），即圆的轮廓颜色。
- `setfillcolor` 设置**填充颜色**，即圆的内部是黄色。

EasyX 支持用 `RED`、`GREEN`、`BLUE` 等预设常量，也可以用 `RGB(255, 0, 0)` 来自定义任意颜色。

---

**`fillcircle(400, 300, 150);`**

- `fillcircle` = Fill Circle（填充的实心圆）。EasyX 还有一个 `circle` 函数，画空心圆。
- 参数：`(圆心 x 坐标, 圆心 y 坐标, 半径)`。
- 注意：EasyX 的坐标系原点在**左上角**（`y` 轴朝下），这和数学课上学的不一样。所以 `(400, 300)` 是窗口的正中心（800/2, 600/2）。

> **停下来想一想**：如果你用的是数学课上学的坐标系（原点在左下角，y 轴朝上），`(400, 300)` 会在窗口的哪个位置？这个差异重要吗？答案是：屏幕坐标系的原点确实在左上角，几乎所有图形编程都遵守这个约定。你需要习惯的是——`y` 值越大，位置越靠下。

---

**`outtextxy(280, 280, _T("Hello Red Circle!"));`**

- `outtextxy` = Output Text at XY，在指定坐标位置输出文字。
- `_T(...)` 是一个宏（Macro），让中文字符串和英文字符串都能兼容处理。如果你只写英文，可以省略 `_T`。
- 注意文字覆盖在圆上面——EasyX 的绘制是有**叠放顺序**的：后画的盖在先画的上面。

---

**`getchar();`**

这行代码的作用是"等我按任意键"。如果没有它，程序画完圆的一瞬间就执行 `closegraph()` 关闭窗口了，你根本来不及看清。`getchar()` 让程序在这里停下来，等你按一下键盘。

---

**`closegraph();`**

关闭图形窗口，释放它占用的系统资源。好习惯：开了窗口就要关。

---

> **动手试一下**：修改下面的参数，每次改完按 F5 运行，看看效果：
> 1. 把圆的半径从 150 改成 80
> 2. 把背景色 `BLACK` 改成 `WHITE`，填充色 `YELLOW` 改成 `BLUE`
> 3. 把窗口大小改成 `initgraph(1024, 768)`
> 4. 把文字的坐标改到圆心正上方
> 5. 在圆旁边再画一个不同颜色的圆（用 `fillcircle` 再加一行）

## 0.4 理解发生了什么（约 30 分钟）

上一节你运行了一个程序，看到了一个圆。这一节我们退一步，理解在你按下 F5 之后的几秒钟里，电脑到底干了什么。

### 0.4.1 从源代码到可运行程序：编译和链接

你写的 `main.cpp` 文件里那些英文和标点符号，叫做**源代码**（Source Code）。电脑的 CPU 不认识这些文字——它只认识由 0 和 1 组成的**机器指令**（Machine Code）。

把源代码变成机器指令需要两步：

```
源代码 (main.cpp) ──编译──> 目标文件 (main.obj) ──链接──> 可执行文件 (Project1.exe)
                      ↑ 编译器 (cl.exe)                 ↑ 链接器 (link.exe)
```

**编译（Compile）**：编译器逐行读你的代码，检查语法，如果没错误，就把它翻译成机器语言，生成一个叫 `main.obj` 的**目标文件**（Object File）。这个文件已经是机器指令了——但是它不完整，因为它还不知道 `initgraph` 之类的函数到底在哪。

**链接（Link）**：链接器把 `main.obj` 和 EasyX 的库文件（`easyx.lib`，里面是 EasyX 所有函数的机器指令实现）以及 C++ 标准库的代码"拼"在一起，最终生成一个 `.exe` 文件——这就是你能双击运行的程序。

在 Visual Studio 里，按 F5 会自动完成编译 + 链接 + 运行。如果中间出错，VS 会在窗口底部的"输出"面板里告诉你错在哪一行。

### 0.4.2 像素、坐标、和画布

电脑屏幕是由极小的发光点组成的，这些点叫**像素**（Pixel）。一个 1920x1080 分辨率屏幕意味着横向有 1920 个像素，纵向有 1080 个像素，总共约 200 万个像素。

当你写 `initgraph(800, 600)` 时，EasyX 创建了一个 800 像素宽、600 像素高的窗口。每一个像素都有一个坐标 `(x, y)`：

```
(0,0) ──────────────────────── x 轴 → (799, 0)
  │  (100, 100) 在这里
  │
  │              (400, 300) 是窗口正中心
  │
  y 轴
  ↓
(0, 599) ──────────────────── (799, 599)
```

> **停下来想一想**：如果窗口是 800x600，那最右下角的坐标是多少？答案是 `(799, 599)`，不是 `(800, 600)`——因为坐标从 0 开始数，0 到 799 正好是 800 个像素。

### 0.4.3 程序执行的顺序

C++ 程序是**从上到下、一行一行**依次执行的（这是简化说法，后面学了函数调用和循环之后会更复杂）。

在我们的程序里：
1. 第 10 行 `initgraph` 先执行——创建窗口
2. 第 13-14 行设置背景并清屏
3. 第 17 行设置线条颜色
4. 第 20 行设置填充颜色
5. 第 24 行画圆
6. 第 27-29 行设置文字样式并显示文字
7. 第 32 行 `getchar` 等待按键——程序暂停
8. 按任意键后，第 35 行 `closegraph` 关闭窗口
9. 第 37 行 `return 0` 结束程序

这就是**顺序执行**的概念。你现在写的所有画图代码，执行顺序就是它们出现的顺序。如果你把 `getchar()` 放到 `initgraph` 前面——程序还没开窗口就卡在那等你按键了。

### 0.4.4 一些关键概念的速查

| 术语 | 含义 | 类比 |
|------|------|------|
| **源代码** | 你写的 `.cpp` 文件里的文字 | 菜谱 |
| **编译器** | 把源代码翻译成机器指令的工具 | 翻译官 |

## 常见问题 FAQ

### Q: VS 装到一半卡住了，进度条不动了？
**A:** 这通常是因为它在下载大文件（SDK 包有几个 GB）。如果超过 30 分钟完全不动，重启安装程序。Windows 更新也可能会干扰安装——可以先手动暂停 Windows 更新再装 VS。

### Q: EasyX 安装程序检测不到我的 VS？
**A:** 首先确保你的 VS 版本是 VS 2022 Community **正式版**而非 Preview 版。其次，确保 VS 安装时确实勾选了"使用 C++ 的桌面开发"（不是其他工作负载）。如果都正确，试试以管理员身份运行 EasyX 安装程序。

### Q: 编译报错 `无法打开包括文件: "graphics.h"`？
**A:** 99% 的情况是 EasyX 没装对。重新运行 EasyX 安装程序，确认 VS 版本前面的复选框是勾上的状态。

### Q: 运行后窗口闪了一下就没了？
**A:** 你没有加 `getchar()` 或类似等待按键的代码。程序画完圆就直接执行 `closegraph()` 退出了。加上 `getchar()` 或 `system("pause")` 即可。

### Q: 我的 VS 主界面是英文的，能改成中文吗？
**A:** VS 2022 安装时就让你选了语言。如果已经装完了，打开 Visual Studio Installer，在 VS 2022 那一行点"修改"，切换到"语言包"标签页，勾选"中文（简体）"，点安装。

### Q: Mac 怎么办？
**A:** Visual Studio Community 只有 Windows 版。Mac 用户有三个选项：(1) 装双系统或虚拟机（推荐 Parallels Desktop）在里面装 Windows + VS；(2) 使用学校/公司的 Windows 电脑；(3) VS for Mac 不支持 C++ 桌面开发，不要下。如果你实在不想碰 Windows，可以考虑等价方案：VS Code + Clang/GCC + SDL2，但这个课程的所有截屏和指令都是基于 VS 的，自己折腾环境可能会多花不少时间。

### Q: 能用 VS 2019 或更老的版本吗？
**A:** 可以，EasyX 支持 VS 2010 到 VS 2022。但建议用 VS 2022，因为它的编译器对 C++ 标准的支持更好，课程中的代码在新编译器上出错的概率更低。
